📊 Capítulo 8 — Dashboards e Storytelling com Dados

“Dados bem apresentados contam uma história. Dados mal apresentados... confundem todo mundo.”

🎯 O que é um Dashboard?

Um dashboard é um painel visual que resume dados importantes e transforma planilhas e tabelas em insights rápidos. Ele é o ponto onde a análise encontra o design — e onde números passam a fazer sentido.

💡 Definição simples: Dashboard é um painel interativo que traduz dados em decisões.

🧩 Tipos de Dashboards

Tipo Objetivo Exemplo
Operacional Monitora métricas em tempo real. Pedidos por minuto, status de sistemas.
Tático Acompanha metas e indicadores de médio prazo. Desempenho semanal de vendas.
Estratégico Ajuda na tomada de decisão de longo prazo. Indicadores de crescimento, churn, receita.
🎨 Dica: Antes de criar o dashboard, defina “para quem” e “para quê” ele serve.

🏗️ Etapas da Criação de um Dashboard

  1. Defina o objetivo (qual pergunta o dashboard deve responder?)
  2. Escolha as métricas (KPIs) certas
  3. Limpe e modele os dados
  4. Escolha a ferramenta (Power BI, Google Data Studio, Streamlit...)
  5. Projete o layout visual
  6. Teste e otimize a experiência do usuário
🧠 Desafio: Monte um esboço no papel do seu dashboard ideal. Quais gráficos e informações realmente ajudam a tomar decisões?

🎨 Princípios de Design de Informação

Inspirado nos mestres Edward Tufte e Cole Knaflic, aqui estão as regras de ouro:

  • 🚫 Evite poluição visual — menos é mais
  • 🎯 Cada elemento deve ter propósito
  • 🎨 Use cores com intenção (não por estética)
  • 🧩 Destaque o que importa — contraste e hierarquia
  • 📖 Construa uma narrativa — inicie com contexto, mostre o problema e conclua com insight
💬 Regra de ouro: “Se um gráfico precisa de legenda para ser entendido, ele pode ser melhorado.”

📈 Gráficos e Quando Usar

Gráfico Uso Ideal Evite se...
Colunas Comparar categorias Tiver mais de 10 categorias
Linhas Mostrar tendências ao longo do tempo Tiver menos de 3 pontos de tempo
Pizza Mostrar proporções Tiver mais de 6 fatias
Dispersão Visualizar correlações Os eixos não forem numéricos
Mapa de Calor Destacar intensidade de valores As cores não tiverem contraste suficiente

⚙️ Ferramenta 1: Power BI

O Power BI é a ferramenta mais popular de BI (Business Intelligence). Ele permite conectar bancos de dados, planilhas e APIs, criar gráficos interativos e publicar dashboards online. 


# Passos básicos no Power BI:
1. Conecte suas fontes (Excel, SQL, Web, API)
2. Trate e modele os dados no Power Query
3. Crie relações entre tabelas
4. Construa visuais no editor
5. Publique no Power BI Service (para web)
💡 Dica: Use segmentações visuais e cartões de KPI para tornar o painel interativo.

🌐 Ferramenta 2: Google Data Studio (Looker Studio)

O Data Studio (agora Looker Studio) é uma opção gratuita e online para criar dashboards com integração direta ao Google Sheets, BigQuery e outras fontes. 


# Exemplo de fluxo:
1. Conecte um Google Sheets com suas métricas
2. Escolha “Adicionar gráfico” → “Série temporal”
3. Adicione filtros interativos (por data, categoria, etc.)
4. Publique o link e compartilhe com permissões
🔍 Extra: Você pode embutir dashboards do Looker Studio em sites e portais internos.

🐍 Ferramenta 3: Streamlit (Python)

O Streamlit é uma biblioteca Python que transforma scripts em dashboards web interativos, ideais para cientistas de dados. 


import streamlit as st
import pandas as pd
import matplotlib.pyplot as plt

st.title("📊 Dashboard de Vendas")

# Dados simulados
dados = pd.DataFrame({
    "Mês": ["Jan", "Fev", "Mar", "Abr", "Mai"],
    "Vendas": [100, 120, 150, 130, 180]
})

# Gráfico
fig, ax = plt.subplots()
ax.plot(dados["Mês"], dados["Vendas"], marker="o")
st.pyplot(fig)

st.metric("Vendas Totais", sum(dados["Vendas"]))
💡 Vantagem: Ideal para prototipar dashboards de Machine Learning e automações de relatórios.

📖 Storytelling com Dados

Visualização sem contexto é só decoração. O verdadeiro poder dos dados vem quando você **conta uma história com propósito**.

  1. Apresente o contexto — o “porquê” da análise
  2. Mostre o conflito — o problema ou insight
  3. Construa a tensão — explore o que os dados revelam
  4. Entregue a resolução — a resposta que o público buscava
  5. Finalize com ação — o que deve ser feito agora?
🧠 Desafio: Pegue uma planilha comum e crie uma história visual:
  • Qual é o problema?
  • Qual insight você descobriu?
  • Qual ação você recomenda?

🧠 Dicas Avançadas

  • 🎯 Mantenha um foco por página
  • 🧩 Use hierarquias (título → métrica → detalhe)
  • 🎨 Paleta: use no máximo 5 cores — e evite saturadas
  • 📏 Mantenha alinhamentos e espaçamentos consistentes
  • ⚙️ Pense como UX designer: menos cliques = melhor experiência

📊 Boas Práticas de Dashboard

Faça Evite
Usar cores neutras e destaque para alertas Fundo preto com texto colorido demais
Contar uma história com início, meio e fim Mostrar todos os dados de uma vez
Usar métricas de negócio Encher de dados irrelevantes
Testar em telas diferentes (mobile, desktop) Assumir que todo mundo tem o mesmo monitor

🏁 Conclusão

Um bom dashboard não é só bonito — ele é funcional, claro e convincente. É a tradução visual de toda a jornada que você percorreu desde o primeiro dado bruto até o insight final.
